# 🔍 模型响应诊断指南

## 📋 目的

通过详细的日志记录，诊断工具调用失败的根本原因：
- **模型响应问题** - LLM 没有正确生成工具调用
- **提示词问题** - 系统提示词未正确传递或缺失关键信息

---

## 🚀 使用方法

### 方法 1: 使用诊断工具（推荐）

```bash
python test_model_diagnostic.py
```

**功能**:
- 自动设置 DEBUG 日志级别
- 提供交互式测试界面
- 显示详细的诊断信息汇总

### 方法 2: 直接运行测试

```bash
python test_responsiveness.py
```

然后查看日志文件。

## 📝 日志分析要点

### 1. 查看 LLM 响应分析

在日志中查找：
```
[Iteration 1] LLM 响应分析:
  - 响应内容预览：...
  - 工具调用：list_directory({'path': '.'})
  ✅ 工具调用成功
```

**正常情况**:
- ✅ 有工具调用记录
- ✅ 参数完整

**异常情况**:
- ❌ `⚠️ 无工具调用`
- ❌ `❌ 严重问题：既无响应内容也无工具调用！`

### 2. 检查系统提示词

在日志中查找：
```
  - 系统提示词长度：1234 字符
  - ✅ 系统提示词包含 CRITICAL RULES
  - ✅ 系统提示词包含工具使用说明
```

**正常情况**:
- ✅ 包含 CRITICAL RULES
- ✅ 包含 USE TOOLS 相关说明

**异常情况**:
- ❌ `❌ 系统提示词缺少 CRITICAL RULES！`
- ❌ `❌ 系统提示词缺少工具使用说明！`

### 3. 检查流式调用过程

在日志中查找：
```
[流式调用] 开始流式调用 LLM
  - 消息历史长度：3
  - 最后消息类型：HumanMessage
  - 最后内容预览：查看当前文件夹内容...

[流式 chunk #5] 检测到工具调用片段:
  - 工具名称：list_directory
  - 参数预览：{'path': '.'}
  - ID: call_abc123

[流式调用] 完成统计:
  - 总 chunk 数：10
  - 收集文本长度：0 字符
  - 检测到工具调用：是
```

**正常情况**:
- ✅ 检测到工具调用片段
- ✅ 参数信息完整

**异常情况**:
- ❌ 无任何工具调用片段日志
- ❌ 参数为空或不完整

### 4. 检查消息历史统计

在日志中查找：
```
[消息历史统计]
  - 总消息数：5
  - 历史工具调用：#1: list_directory, #3: read_file
```

**正常情况**:
- ✅ 有历史工具调用记录

**异常情况**:
- ❌ `⚠️ 历史消息中无任何工具调用`

## 🔍 常见问题诊断

### 问题 1: 模型完全不调用工具

**日志特征**:
```
[Iteration 1] LLM 响应分析:
  - 响应内容预览：我已收到您的请求...
  - ⚠️ 无工具调用
  - ⚠️ 警告：用户请求需要工具，但模型未调用
```

**可能原因**:
1. ❌ 系统提示词缺失或被忽略
2. ❌ 模型温度过高，倾向于聊天而非行动
3. ❌ 提示词不够明确

**排查步骤**:
1. 检查系统提示词日志：
   ```
   - ✅ 系统提示词包含 CRITICAL RULES
   - ✅ 系统提示词包含工具使用说明
   ```
2. 如果提示词正常，可能是模型本身问题

**解决方案**:
- 调整 temperature（建议 0.5-0.7）
- 增强系统提示词的强制性语气
- 添加 Few-shot 示例

### 问题 2: 流式调用检测不到工具

**日志特征**:
```
[流式调用] 完成统计:
  - 总 chunk 数：15
  - 收集文本长度：234 字符
  - 检测到工具调用：否
```

**可能原因**:
1. ❌ LangChain 流式 API 限制（工具调用分散在多个 chunk）
2. ❌ 工具调用信息丢失

**排查步骤**:
1. 检查是否有 chunk 级别的工具调用日志：
   ```
   [流式 chunk #5] 检测到工具调用片段:
   ```
2. 如果完全没有，说明流式确实没检测到

**解决方案**:
- ✅ 已实现混合模式：流式显示 + 完整调用
- 即使流式没检测到，也会重新 invoke 获取完整工具信息

### 问题 3: 系统提示词未正确传递

**日志特征**:
```
  - ❌ 系统提示词缺少 CRITICAL RULES！
  - ❌ 系统提示词缺少工具使用说明！
```

**可能原因**:
1. ❌ 代码修改导致提示词被覆盖
2. ❌ 消息历史被错误重置

**排查步骤**:
1. 检查初始化代码中的 system_prompt
2. 检查是否有地方修改了 self.system_prompt

**解决方案**:
- 确保使用默认的系统提示词
- 不要随意修改 self.system_prompt

### 问题 4: 工具调用参数不完整

**日志特征**:
```
[流式 chunk #5] 检测到工具调用片段:
  - 工具名称：write_file
  - 参数预览：{'path': 'test.txt'}  ← 只有 path，缺少 content
```

**可能原因**:
1. ❌ 参数字符串分散在多个 chunk 中
2. ❌ 解析逻辑有问题

**解决方案**:
- ✅ 已通过重新 invoke 解决此问题
- 检查完整调用后的参数是否完整

## 📊 日志关键字段说明

### LLM 响应分析部分

| 字段 | 说明 | 正常值 |
|------|------|--------|
| 响应内容预览 | AI 返回的文本内容前 500 字符 | 有意义的内容 |
| 工具调用 | 调用的工具列表 | list_directory(...) |
| ⚠️ 无工具调用 | 警告标记 | 不应出现 |
| ❌ 严重问题 | 错误标记 | 绝不应出现 |

### 系统提示词检查部分

| 字段 | 说明 | 期望结果 |
|------|------|----------|
| 系统提示词长度 | 提示词字符数 | >500 |
| CRITICAL RULES | 是否包含关键规则 | ✅ |
| 工具使用说明 | 是否包含工具使用指导 | ✅ |

### 流式调用部分

| 字段 | 说明 | 说明 |
|------|------|------|
| chunk 数 | 流式返回的片段数 | 通常 5-20 |
| 收集文本长度 | 收集的文本总长度 | 可变 |
| 检测到工具调用 | 是否检测到工具 | 是/否 |

## 🎯 快速诊断流程

### Step 1: 运行诊断测试

```bash
python test_model_diagnostic.py
# 选择选项 1: 查看目录内容
```

### Step 2: 查看最新日志

```bash
# Windows
Get-Content logs\Agent_*.log -Tail 100

# Linux/Mac
tail -n 100 logs/Agent_*.log
```

### Step 3: 查找关键日志

按顺序查找：

1. **LLM 响应分析** - 看是否有工具调用
2. **系统提示词检查** - 看提示词是否完整
3. **流式调用统计** - 看流式是否检测到工具
4. **消息历史统计** - 看整体工具调用情况

### Step 4: 判断问题类型

**如果是提示词问题**:
- ❌ 系统提示词缺少 CRITICAL RULES
- ❌ 系统提示词缺少工具使用说明
- → 检查代码中的 system_prompt 定义

**如果是模型响应问题**:
- ✅ 系统提示词完整
- ❌ 但模型仍不调用工具
- → 调整 temperature 或更换模型

**如果是流式问题**:
- ❌ 流式未检测到工具调用
- ✅ 但完整调用成功
- → 正常，混合模式已处理

## 📖 示例分析

### 成功案例

```
2026-03-07 13:00:00 [INFO] Agent: [Iteration 1] LLM 响应分析:
2026-03-07 13:00:00 [INFO] Agent:   - 响应内容预览：(空内容)
2026-03-07 13:00:00 [INFO] Agent:   - 工具调用：list_directory({'path': '.'})
2026-03-07 13:00:00 [DEBUG] Agent:   - ✅ 工具调用成功，无需重试
```

**分析**: 
- ✅ 响应内容为空（正常，因为只需要工具调用）
- ✅ 成功调用 list_directory 工具
- ✅ 一次成功

### 失败案例 - 提示词问题

```
2026-03-07 13:00:00 [WARNING] Agent: [Iteration 1] LLM 响应分析:
2026-03-07 13:00:00 [WARNING] Agent:   - 响应内容预览：好的，我来帮你查看目录内容...
2026-03-07 13:00:00 [WARNING] Agent:   - ⚠️ 无工具调用
2026-03-07 13:00:00 [ERROR] Agent:   - ❌ 系统提示词缺少 CRITICAL RULES！
2026-03-07 13:00:00 [ERROR] Agent:   - ❌ 系统提示词缺少工具使用说明！
```

**分析**:
- ❌ 模型只说不做
- ❌ 系统提示词被修改过
- → 需要恢复默认 system_prompt

### 失败案例 - 模型问题

```
2026-03-07 13:00:00 [WARNING] Agent: [Iteration 1] LLM 响应分析:
2026-03-07 13:00:00 [WARNING] Agent:   - 响应内容预览：这是一个很好的问题...
2026-03-07 13:00:00 [WARNING] Agent:   - ⚠️ 无工具调用
2026-03-07 13:00:00 [WARNING] Agent:   - ⚠️ 警告：用户请求需要工具，但模型未调用
2026-03-07 13:00:00 [DEBUG] Agent:   - ✅ 系统提示词包含 CRITICAL RULES
2026-03-07 13:00:00 [DEBUG] Agent:   - ✅ 系统提示词包含工具使用说明
```

**分析**:
- ✅ 系统提示词完整
- ❌ 模型仍然不调用工具
- → 模型温度可能过高，或模型本身不适合
